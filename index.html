HTML

  • Enter your HTML notes here:
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • It is important to always remember to fill out the metadata in a page.
  • A particularly important point of metadata is the viewport, which sizes the page to fit varied screens.
  • There are many tags in HTML, but all of them use the angular bracket as demarkation.
  • HTML is like the skeleton of a webpage's anatomy.

CSS

  • Enter your CSS notes here:
  • CSS is constructed differently from HTML, but useing some of the same tags.
  • The HTML tags that appear in CSS are being defined or given style structure in the CSS.
  • Once linked, external CSS sheets grant a developer the ability to totally transform the visual quality of their webpage.
  • A tag like h1 would go without any prefix designation in the CSS.
  • A class selector (i.e. '.card') is preceded with a period, which lets the CSS know it is a class.
  • Color in CSS can be done many ways, from hexadecimals to words to hex #.
  • CSS is like the skin, hair, eyes, and other aesthetic external bodyparts in a webpage's anatomy.

Git

  • Enter your Git notes here:
  • Git status: checks what branch we are currently on.
  • Git checkout -b branch-name: creates a new branch and switches to it.
  • Git status can be used to get an idea of what is being done and needs to be committed - use it OFTEN.
  • Cd [path] can be used to navigate through folders in git, just using cd brings you back to the start.
  • The 'touch file.type' allows us to directly create a file of any type through the console.
  • Git is like the neural network of a webpage's anatomy, relaying information about the page between the branches and the main network.

JavaScript

  • Enter your JavaScript notes here:
  • Variables are your building blocks, serving to store data for easy access and to minimize repetition.
  • Comments in JavaScript are the same as those in CSS - ex: /* your comment here */.
  • One-line comments can be done simply as //comment.
  • Operators allow you to do math with your JavaScript, from simple arithmetic to complex boolean comparisons.
  • Conditionals like 'if... else...' allow you to test whether an expression is true or not.
  • Functions, like variables, allow you to cut out tedious repetition and streamline your functional code into manageable building blocks.
  • Events listen for an input and then output code in response - if you've ever clicked on a button in a webpage, you've used one!
  • There are many key data types in JavaScript; Boolean, Undefined, Null, Numeric, Number, NaN, Strings, oh my!
  • A BigInt is created by appending 'n' to the end of an integer or by calling the constructor.
  • Beware of "stringly-typing" your code! Strings are tempting, but can be dangerous! (Just ask the cat!)
  • There is a built-in utility for representing dates, use it!
  • JavaScript is like the muscles of our webpage's body, allowing it to interact more fluidly with the world.
